# 🚀 AI Startup Idea Validator

An AI-powered full-stack web application that analyzes startup ideas and generates comprehensive validation reports in seconds — including market analysis, competitor landscape, tech stack suggestions, risk assessment, and profitability scores.

---

## 📸 Features

- **AI-Powered Analysis** — Submit any startup idea and get a structured JSON report generated by LLaMA 3.3 70B via Groq (free, blazing fast)
- **Profitability Score** — Visual circular gauge showing a 0–100 profitability score
- **Risk Badge** — Color-coded Low / Medium / High risk indicators
- **Competitor Analysis** — 3 named competitors with one-line differentiators
- **Tech Stack Recommendations** — Curated 4–6 technologies for your MVP
- **Market & Customer Insights** — Concise market size overview and ideal customer persona
- **Dashboard** — Browse all previously validated ideas at a glance
- **Delete Ideas** — Remove unwanted reports from the dashboard
- **Persistent Storage** — All reports stored in PostgreSQL

## 🚀 Installation & Setup

### 1. Clone the repository

```bash
git clone with url
cd "Schmooze Media"
```

---

### 2. Backend Setup

#### a. Create and activate virtual environment

```bash
cd Backend
python -m venv venv

# Windows
.\venv\Scripts\activate

# macOS/Linux
source venv/bin/activate
```

#### b. Install dependencies

```bash
pip install -r requirements.txt
```

#### c. Configure environment variables

Create a `.env` file inside the `Backend/` directory:

```env
DATABASE_HOSTNAME=localhost
DATABASE_USERNAME=postgres
DATABASE_PASSWORD=your_postgres_password
DATABASE_NAME=Schmooze_Media
DATABASE_PORT=5432
or 
DATABASE_URL = postgresql://postgres:password:5432/Schmooze_Media
GROQ_API_KEY=your_groq_api_key_here
```

#### d. Create the PostgreSQL database

Connect to PostgreSQL and run:

```sql
CREATE DATABASE "Schmooze_Media";
```

Or via psql CLI:

```bash
psql -U postgres -c 'CREATE DATABASE "Schmooze_Media";'
```

#### e. Start the backend server

```bash
uvicorn main:app --reload
```

The backend will be available at: **http://localhost:8000**
Interactive API docs (Swagger UI): **http://localhost:8000/docs**

---

### 3. Frontend Setup

Open a **new terminal**:

```bash
cd Frontend
npm install
npm run dev
```

The frontend will be available at: **http://localhost:5173**

## 🤖 AI Prompt Design

The AI uses the following structured prompt to ensure consistent JSON output:

```
You are an expert startup consultant. Analyze the given startup idea
and return a structured JSON object with the fields: problem,
customer, market, competitor, tech_stack, risk_level,
profitability_score, justification.

Rules:
- Keep answers concise and realistic.
- 'competitor' should contain exactly 3 competitors with one-line differentiation each.
- 'tech_stack' should be 4–6 practical technologies for MVP.
- 'profitability_score' must be an integer between 0–100.

Return ONLY JSON.
```

**Model Used:** `llama-3.3-70b-versatile` via Groq API (OpenAI-compatible endpoint)

## 📝 Architecture Notes

- **OpenAI-Compatible SDK on Groq:** We use the `openai` Python library pointed at Groq's base URL (`https://api.groq.com/openai/v1`), which avoids adding extra dependencies while leveraging LLaMA 3.3.
- **JSON Report Storage:** The AI report is stored as a PostgreSQL JSON column, allowing flexible querying and future schema evolution without migrations.
- **Error Resilience:** The AI service wraps all calls in try/except, so a failed AI call still saves the idea to the database with a graceful error object in the report field.
- **CORS:** Currently set to `allow_origins=["*"]` for MVP development. Lock this down to your frontend URL in production.
